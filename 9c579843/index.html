Redis 帮助文档

Redis 帮助文档

Tips: 输入help后按下Tab键可自动补全

Redis 数据结构

Redis是一个键值型数据库，key一般为String类型，value类型如下：

类型例子描述
Stringhello world字符串
Hash{name: “jack”, age: 21}哈希表
List[A -> B -> C -> D]有序集合（链表）
Set{A, B, C}无序集合（不能重复）
SortedSet{A: 1, B: 2, C: 3}有序集合（不能重复）
GEO{A: (120.3, 30.5)地理坐标（经纬度）
BitMap0110110101110101011位图
HyperLog0110110101110101011基数统计

其中，String、Hash、List、Set、SortedSet为基本类型，GEO、BitMap、HyperLog为特殊类型。

Redis 通用命令

  • KEYS pattern：查看符合模板的所有key

    [pattern]
    * 代表多个字符
    ? 代表一个字符

    模糊查询，低效率。当数据量规模足够大时，往往会给服务器带来巨大负担；因为Redis为单线程，搜索过程中无法执行其他命令，导致整个Redis服务阻塞。

    不建议在生产环境设备上使用（若为集群模式，在从节点上执行命令，则可行）

  • DEL key [key ...]：删除指定的key

    返回值 (integer) : 删除的key的数量

  • EXISTS key [key ...]：判断key是否存在

    返回值 (integer) : 存在的key的数量

  • EXPIRE key seconds [NX|XX|GT|LT]：给一个key设置有效期，有效期到期时，该key会被自动删除（单位：秒）

  • TTL key：查看一个KEY的剩余有效期

    返回值 (integer) ：-2 已经被移除；-1 永久有效

String 类型

字符串类型，Redis中最简单的存储类型。

根据字符串的不同格式，value可以分为三类

  • String：普通字符串
  • int：整数类型
  • float：浮点类型

整数型和浮点型数据，可以做自增、自减操作。

不管是哪种格式，底层都是字节数据组形式存储，只不过编码方式不同（数值类型将数字转为二进制形式进行存储），字符串类型的最大空间不能超过512MB。

String类型的常见命令

  • 基础命令

    • SET key value [NX|XX] [GET] [EX seconds|PX milliseconds|EXAT unix-time-seconds|PXAT unix-time-milliseconds|KEEPTTL]：添加或修改已经存在的一个String类型的键值对
    • GET key：根据key获取String类型的value
    • MSET key value [key value ...]：批量添加多个String类型的键值对
    • MGET key [key ...]：根据多个key获取多个String类型的value

  • 数值类型

    • INCR key：让一个整型key自增1

    • INCRBY key increment：让一个整型的key自增并指定步长

      自减：DECR 或 INCRBY key -1

    • INCRBYFLOAT key increment：让一个浮点类型的数字自增并指定步长

  • 组合命令

    • SETNX key value：添加一个String类型的键值对，前提是这个key不存在，否则不执行

      相当于SET key value NX

    • SETEX key seconds value：添加一个String类型的键值对，并指定有效期

      相当于SET key value EX seconds

Key的层级格式

Redis的key允许有多个单词形成层级结构，多个单词之间用 ‘:’ 隔开，格式：[项目名]:[业务名]:[类型]:[id]

这个格式并非固定，也可以根据自己的需求来删除或添加词条。

如果value是一个Java对象，则可以将对象序列化为JSON字符串后存储。

KEYVALUE
pro:user:1{“id”:1, “name”:”JACK”, “age”:21}
pro:product:1{“id”:1, “name”:”mi”, “price”:4999}

Hash 类型

散列，其value是一个无序字典，类似于Java中的HashMap结构。

String类型的缺点：将对象序列化为JSON字符串后存储，当需要修改对象某个字段时不方便。

Hash结构可以将对象中的每个字段独立存储，可以针对单个字段做CRUD。

Hash类型的常见命令

  • HSET key field value [field value ...]：添加或者修改hash类型key的field的值
  • HGET key field：获取一个hash类型key的field的值

  • HMSET key field value [field value ...]：批量添加多个hash类型key的field的值

    注：从Redis 4.0.0版本开始，这条命令被弃用。在迁移或编写新的代码时，它可以被带有多个字段-值对的HSET取代。

  • HMGET key field [field ...]：批量获取多个hash类型key的field的值
  • HGETALL key：获取一个hash类型的key中的所有的field和value
  • HKEYS key：获取一个hash类型的key中的所有的field
  • HVALS key：获取一个hash类型的key中的所有的value
  • HINCRBY key field increment：让一个hash类型key的字段值自增并指定步长
  • HSETNX key field value：添加一个hash类型的key的field值，前提是这个field不存在，否则不执行

List 类型

Redis中的List类型与Java中的LinkedList类似，可以看作是双向链表结构（实际更复杂）。既可以支持正向检索，也可以支持反向检索。

特征也和LinkedList类似：

  • 有序
  • 元素可重复
  • 插入和删除快
  • 查询速度一般

常用于存储有序数据，如：朋友圈点赞列表和评论列表等。

List类型的常见命令

  • 队首（左侧）

    • LPUSH key element [element ...]：向列表左侧插入元素
    • LPOP key [count]：移除并返回列表左侧的第一个元素，没有则返回nil

  • 队尾（右侧）

    • RPUSH key element [element ...]：向列表右侧插入一个或多个元素
    • RPOP key [count]：移除并返回列表右侧的第一个元素
  • LRANGE key start stop：返回一段角标范围内的所有元素

  • BLPOP key [key ...] timeout和BRPOP key [key ...] timeout：类似于LPOP和RPOP，但会在没有元素时等待指定时间。

    注：B = Block 阻塞

    案例：

  • 如何用List结构模拟一个栈？

    • 入口和出口在同一边
    • 类比：喝酒喝吐了，后面喝进去的先吐出来

  • 如何用List结构模拟一个队列？

    • 入口和出口在不同边
    • 类比：吃完上厕所，先吃进去的先排出来

  • 如何利用List结构模拟一个阻塞队列？

    • 入口和出口在不同边
    • 出队时采用BLPOP或BRPOP

Set 类型

Redis的Set结构和Java中的HashSet类似，可以看作是一个value为null的HashMap。

特征也和HashSet类似：

  • 无序
  • 元素不重复
  • 查询速度快
  • 支持交集、并集、差集等功能

Set类型的常见命令

  • SADD key member [member ...]：添加指定元素
  • SREM key member [member ...]：移除指定元素
  • SCARD key：返回set中的元素个数
  • SISMEMBER key member：
  • SMEMBERS key：判断一个元素是否存在于set中
  • SINTER key [key ...]：求交集
  • SDIFFkey [key ...]：求差集
  • SUNION key [key ...]：求并集

SortedSet 类型

Redis的SortedSet是一个可排序的set集合，与Java中的TreeSet有些类似，但底层数据结构却差别很大。SortedSet中的每一个元素都带有一个score属性，可以基于score属性对元素排序，底层的实现是一个跳表 (SkipList) 加hash表。

SortedSet具备以下特性：

  • 可排序
  • 元素不重复
  • 查询速度快

因为SortedSet的可排序特性，经常被用来实现排行榜这样的功能。

SortedSet类型的常见命令

  • ZADD key [NX|XX] [GT|LT] [CH] [INCR] score member [score member ...]：添加元素到sorted set，如果已经存在则更新其score值
  • ZREM key member [member ...]：删除sorted set中的一个指定元素
  • ZSCORE key member：获取sorted set中的指定元素的score值
  • ZRANK key member：获取sorted set中的指定元素的排名
  • ZCARD key：获取sorted set中的元素个数
  • ZCOUNT key min max：统计score值在给定范围内的所有元素的个数
  • ZINCRBY key increment member：让sorted set中的指定元素自增，步长为指定的increment值
  • ZRANGE key start stop [BYSCORE|BYLEX] [REV] [LIMIT offset count] [WITHSCORES]：按照score排序后，获取指定排名范围内的元素
  • ZRANGEBYSCORE key min max [WITHSCORES] [LIMIT offset count]：按照score排序后，获取指定score范围内的元素
  • ZDIFF numkeys key [key ...] [WITHSCORES], ZINTER numkeys key [key ...] [WEIGHTS weight [weight ...]] [AGGREGATE SUM|MIN|MAX] [WITHSCORES], ZUNION numkeys key [key ...] [WEIGHTS weight [weight ...]] [AGGREGATE SUM|MIN|MAX] [WITHSCORES]：求差集、交集、并集

注：所有的排名默认为升序，若要降序则在Z后面添加REV。(REV: reverse 反转)
