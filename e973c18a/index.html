数据关联关联的 (Relational)无关联的
查询方式SQL 查询非 SQL
事务特性ACIDBASE

扩展性垂直水平
使用场景1) 数据结构固定
2) 相关业务对数据安全性，一致性要求较高		1) 数据结构不固定
2) 相关业务对数据对一致性、安全性要求不高
3) 对性能要求较高

SQL

Structured Query Language

关系型数据库

  1. 结构化 (Structured)

    数据有固定的格式要求，如主键约束，唯一约束，无符号约束，数据类型，数据长度等约束。

    一般不建议修改表的结构。

  2. 关联的 (Relational)

    数据与数据之间往往有关联，如用户表，商品表和订单表，通过外键联系，节省数据存储空间。

  3. SQL查询

    语法固定，只要是关系型数据库，都能够使用相同的语句进行查询。

    SQL: SELECT id, name, age FROM tb_user WHERE id = 1

  4. ACID

    数据的增删改查，事务需求，满足ACID特性：

    • 原子性 (atomicity)
    • 一致性 (consistency)
    • 隔离性 (isolation)
    • 持久性 (durability)

NoSQL

非关系型数据库

  1. 非结构化

    字段约束松散，没有强约束，数据结构没有严格要求。

    • 键值类型 Redis
    • 文档类型 MongoDB
    • 列类型 HBase
    • Graph类型 Neo4j

  2. 无关联的

    数据与数据之间没有直接维护的关联 (外键)，常见的是json文档以嵌套的形式进行维护。

  3. 非SQL

    语法格式不统一

    Redis: get user:1

    MongoDB: db.users.find({_id: 1})

    elasticsearch: GET http://localhost_9200/users/1

  4. BASE

    无法满足事务的强一致性，只有一些基本的一致性：

    基本可用 (Basically Available)

    软状态 (Soft state)

    最终一致性 (Eventually consistent)

认识 Redis

Redis诞生于2009年，全称是Remote Dictionary Server，远程词典服务器，是一个基于内存的键值型NoSQL数据库。

特征：

  • 键值 (key-value) 型，value支持多种不同数据结构，功能丰富
  • 单线程，每个命令具备原子性
  • 低延迟、速度快（基于内存、IO多路复用、良好的编码）
